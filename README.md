# Как приручить агента

Дружеский обучающий челлендж по созданию **end-to-end BI агента**.  
Одни данные, одна задача, тюним агента, смотрим результат, учимся.

---

### Стек
* **Агент:** Любой на ваш выбор (Claude, Cursor, GPT-4o, Windsurf и др.).
* **Database:** Postgres.
* **Transformation:** dbt.
* **BI Layer:** BI-as-a-code (Lightdash, Superset, Evidence, Rill и прочие).

## 🤝 Как стать участником (Collaborator)
Поскольку репозиторий публичный, для создания веток вам нужны права на запись:

1. **Оставьте заявку:** Напишите свой GitHub-ник в **Issues** этого репозитория.
2. **Примите приглашение:** GitHub пришлет уведомление на почту. Также его можно найти здесь: `https://github.com/ваш-логин/название-репозитория/invitations`.
3. **Начинайте:** Только после принятия приглашения вы сможете сделать `git push` своей ветки.
---

## 🛠 Правила репозитория (Workflow)
В этом проекте мы используем специфическую структуру веток:

1.  **Main — это фундамент.** В этой ветке лежит только базовая инфраструктура и исходные данные.
2.  **Агенты и их версии живут в ветках.**  
3.  **No Merges.** Мы **никогда не вливаем** ветки решений в `main`. 
4.  **Нейминг:** Называйте свою ветку по шаблону: `[username]-[agent]_[version]`  
    *Пример: `ivanov-cursor_v1`*

---

## 📜 Ограничения
* **Без самописного JS:** Без d3.js, custom charts и прочего хардкода. 
* **Только заданные данные:** Используем только те данные, что есть в `main`
* **Воспроизводимость:** Решение должно запускаться одной командой (через Docker или скрипт).

---

## 📝 Что должно быть в вашей ветке
Помимо кода, в корне вашей ветки должен появиться файл `Result.md` со следующим контентом:
1.  **Инструмент:** Какую модель и среду использовали.
2.  **Лог промптов:** Ключевые команды, которые вы давали агенту.
3.  **Сложности:** На чем агент "тупил" или где пришлось исправлять его руками.
4.  **Результат:** Скриншот итогового дашборда.

---

## 🚀 Как начать
1.  Сделайте `git clone`.
2.  Создайте свою ветку: `git checkout -b yourname-agentname_version`.
3.  Разверните окружение:

Для работы используйте **Colima** (Mac) или **WSL2** (Windows). 
Выберите ОДИН инструмент, который вам нравится, и запустите его вместе с базой данных:

| Инструмент | Фишка | Команда запуска | Порт |
| :--- | :--- | :--- | :--- |
| **Lightdash** | Идеально для dbt | `docker-compose --profile lightdash up -d` | `8080` |
| **Evidence** | Markdown + SQL | `docker-compose --profile evidence up -d` | `3000` |
| **Metabase** | Просто и быстро | `docker-compose --profile metabase up -d` | `3001` |
| **Superset** | Мощный Open Source | `docker-compose --profile superset up -d` | `8088` |
| **Grafana** | Для динамики | `docker-compose --profile grafana up -d` | `3002` |
| **Или добавьте свой** | |  |  |

*Чтобы выключить всё и очистить ресурсы:* `docker-compose down --remove-orphans`

4. Приручайте своего агента 🐲
